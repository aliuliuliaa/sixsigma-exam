RX3i 学习指南

PACSystems RX3i · PME · Cimplicity

学习路线图
1
入门 硬件基础（1-2周）
  • 了解 PACSystems 家族：RX3i/RX7i/RSTi-EP 的区别和定位
  • 掌握背板系统：通用背板 vs 串行扩展背板，PCI 双总线概念
  • 认识 CPU 型号：CPE305/310/330/400 的性能差异和选型
  • 学习电源选型：功率计算，冗余电源配置
  • 了解 I/O 模块：DI/DO/AI/AO/HSC 模块的型号和接线
  • 硬件安装：背板组装、模块安装、接线规范
目标：能看懂硬件清单，理解系统架构图，知道每个模块是干什么的。
了解 PACSystems 家族产品线
掌握通用背板和串行背板的区别
能区分各 CPU 型号的性能参数
了解常用 I/O 模块型号和接线方式
理解电源功率计算和选型
2
进阶 PME 编程（2-4周）
  • 安装 PME：软件安装、授权、界面认识
  • 新建项目：选择 Target（RX3i CPU）、硬件配置
  • 梯形图编程 (LD)：触点、线圈、定时器、数学指令、跳转/调用
  • 结构化文本 (ST)：IF/CASE/FOR/WHILE 语句、函数定义
  • 变量管理：变量声明、地址映射、断电保持设置
  • 子程序和函数块：模块化编程思想
  • 在线调试：下载/上传、监控、强制、断点
  • 系统变量 %S：常用系统状态变量的应用
建议边学边练：在 PME 仿真器中搭建一个简单项目（如电机启停控制），逐步增加功能。
如果没有实际 PLC，可以安装 PME 的仿真功能进行离线调试。
能独立新建项目和配置硬件
掌握梯形图基本指令
能用 ST 编写数据处理逻辑
掌握子程序调用和参数传递
能使用在线调试功能（监控/强制/断点）
完成一个综合练习项目（如水箱液位控制）
3
高级 通信网络（1-2周）
  • Ethernet 基础：IP 地址配置、Ping 测试
  • SRTP 通信：PME 与 CPU 连接、CPU 间数据交换
  • Modbus TCP：主站/从站配置、地址映射
  • EGD 广播：多 PLC 间数据共享配置
  • OPC UA：Server 配置、安全设置、与上位机连接
  • PROFINET：PNC001 模块配置、IO 设备连接（可选）
通信是实际工程中的重点和难点，建议至少精通 SRTP 和 Modbus TCP 两种。
配置 CPU 以太网口 IP 和通信参数
通过 PME 远程连接 CPU 下载/调试程序
配置 Modbus TCP 从站并测试通信
了解 EGD 广播通信的配置方法
配置 OPC UA Server 并连接客户端测试
4
应用 Cimplicity 上位机（2-3周）
  • 软件安装：Cimplicity 安装、授权
  • 新建工程：创建项目、配置设备（连接 PLC）
  • 点位配置：新建 Point、地址映射、采集周期
  • 画面开发：CimEdit 画图、动画绑定、脚本编写
  • 报警配置：报警分类、报警级别、报警消息
  • 趋势图：实时趋势、历史趋势配置
  • 用户管理：权限分级、登录验证
  • 项目部署：工程发布、运行测试
Cimplicity 的学习重点是画面开发 + 点位配置 + 报警系统，这三项覆盖了 80% 的日常使用场景。
安装 Cimplicity 并新建工程
配置 PLC 设备连接（SRTP）
批量配置点位并验证数据采集
开发一个完整的监控画面（含动画+脚本）
配置报警系统并测试
配置趋势图（实时+历史）
练习项目建议

练习 1：电机启停控制（入门）

  • 硬件：1个 DI（启动按钮）、1个 DI（停止按钮）、1个 DO（电机接触器）
  • 程序：自锁电路 + 过载保护 + 运行指示
  • 上位机：启停按钮 + 运行指示灯 + 故障报警

练习 2：水箱液位控制（进阶）

  • 硬件：1个 AI（液位变送器）、1个 AO（调节阀）、1个 DI（进料泵）
  • 程序：PID 闭环控制（用 ST 写 PID 逻辑或用功能块）
  • 上位机：液位棒图 + 阀位显示 + 趋势曲线 + 高低限报警

练习 3：多设备通信（高级）

  • 两台 RX3i 通过 SRTP 交换数据
  • 一台 RX3i 通过 Modbus TCP 读取第三方设备
  • OPC UA 客户端读取 RX3i 数据
  • Cimplicity 同时采集两台 PLC 数据

重点知识速记

⭐ 必记知识点

  • 内存区域：%I(输入) %Q(输出) %M(内部) %AI(模拟入) %AQ(模拟出) %R(寄存器) %T(定时器) %S(系统)
  • 五种语言：LD(梯形图) ST(结构化文本) FBD(功能块) IL(指令表) SFC(顺序图)
  • CPU 模式：STOP / RUN / RUN-IO-Disabled / STOP-IO-Enabled
  • 通信协议：SRTP(私有) Modbus TCP(标准) EGD(广播) OPC UA(未来趋势)
  • 背板总线：PCI(高速 IC695) + 串行(IC694) 双总线可混用
  • 首次扫描：%S0034 仅第一个扫描周期为 ON，用于初始化
  • 1秒脉冲：%S0005 用于闪烁指示灯和定时

⭐ 常见面试/考试题

  • Q: PAC 和 PLC 的区别？
    A: PAC 融合了 PLC 的确定性控制、DCS 的系统管理、PC 的开放性，支持多种编程语言和标准通信协议。
  • Q: RX3i 通用背板的特点？
    A: 双总线设计，PCI 总线支持 IC695 模块，串行总线支持 IC694 模块，可在同一背板上混用。
  • Q: PME 支持哪些编程语言？
    A: LD、ST、FBD、IL、SFC 五种 IEC 61131-3 标准语言。
  • Q: SRTP 和 OPC UA 的区别？
    A: SRTP 是 GE 私有协议，仅限 GE 设备间通信；OPC UA 是工业标准，支持跨平台跨厂商。
  • Q: Cimplicity 的点位采集方式？
    A: 通过 Device 配置连接 PLC，每个 Point 映射到 PLC 地址，按设定的 Scan Rate 周期性读取。
