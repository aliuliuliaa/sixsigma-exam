CIMPLICITY 平台简介 概念

什么是 CIMPLICITY

CIMPLICITY 是 GE Vernova（原 GE Digital）旗下的 Proficy 系列 HMI/SCADA 软件，是全球最大制造商和公用事业公司的首选 HMI/SCADA 之一。从单机到跨厂区的企业级应用，CIMPLICITY 提供真正的客户端/服务端可视化与控制。

核心能力：

  • 集中化配置与授权，集成咨询分析（Advisory Analytics）
  • 原生 HTML5 支持，通过 Proficy Operations Hub 实现移动端和云端访问
  • 面向对象的类与对象设计，实现快速可复用的应用开发
  • OPC UA Server 支持建模数据，聚合系统统一连接
  • 高可用性架构，支持 24/7 不间断运行
  • SCADA Analytics Starter Runtime Edition（基于 CSense 的 AI/ML 模板）

系统架构

CIMPLICITY 采用客户端/服务端架构，核心组件包括：

组件说明
CIMPLICITY Server运行项目配置，与 PLC/控制器通信，管理点位数据、报警、事件等
CIMPLICITY Viewer/Client运行 CimView 显示画面，操作员交互界面（HMI/操作站）
Workbench工程开发环境，创建/修改/运行项目配置
CimEdit画面编辑器，开发 CimView 显示画面
WebSpace远程访问 CimView 画面，通过 Web 浏览器查看
Configuration Hub集中化配置多个 Proficy 产品的 Web 门户
部署模式

单机方案：Server 和 Client 在同一台计算机上，适用于小型单操作员场景。

Server+Client 方案：一台 Server + 多台 Client，所有通信在控制系统区域内，通过防火墙隔离。

带 DMZ 的方案：在控制系统区域与支持 DMZ 之间引入防火墙，支持安全维护。

📋 工程创建与 Workbench

Workbench 工作台 工具

Workbench 概述

Workbench 是 CIMPLICITY 的核心工程开发环境，用于创建、修改和运行项目。所有项目配置（点位、画面、报警、通信等）都在 Workbench 中完成。

Workbench 界面结构：

  • 左侧导航窗格：以树形结构显示项目所有配置分类（Points、Screens、Alarms、Devices 等）
  • 右侧内容窗格：显示当前选中分类的配置记录列表
  • 菜单栏与工具栏：文件操作、编辑、视图、工具等

项目创建流程

  1. 打开 Workbench → File → New Project
  2. 输入项目名称，选择项目路径
  3. 配置项目属性（安全设置、通信选项等）
  4. 添加计算机（Computer）到项目中
  5. 配置资源（Resource）
  6. 创建点位、画面、报警等配置
项目结构

一个 CIMPLICITY 项目包含整个 ICS 或其子集的监控配置。CIMPLICITY 支持多个项目同时运行，可按功能区域划分项目。

项目文件夹结构包括：alarm_help、arc（归档）、data、lock、log、master、pki、RCO、Recipes、screens、scripts、SPC 等。

项目管理操作

操作说明
切换项目同一 Workbench 中切换不同项目
重命名项目修改项目名称（需停机）
拖拽项目元素从一个项目拖拽配置到另一个项目
导出/导入导出配置文件，导入到其他项目
点位系统（Points） 概念

点位概念

Point（点位）是 CIMPLICITY 中最基本的数据单元，代表从 PLC 或其他数据源采集的一个变量。每个点位有唯一的 ID 和配置属性。

点位类型

类型说明示例
Analog（模拟量）连续数值，支持工程量转换温度、压力、流量
Digital（数字量）0/1 状态值电机运行/停止、阀门开/关
Text（文本量）字符串值产品编号、批次号
Device（设备点位）直接映射 PLC 地址%R1、%AI10
Virtual（虚拟点位）仅内存，不映射设备中间计算结果

点位属性配置

  • Point ID：唯一标识符，如 TIC_101_PV
  • Resource：所属资源（关联设备）
  • Device ID：关联的通信设备
  • Address：PLC 内存地址（如 %R100、%AI10）
  • Data Type：INT、UINT、REAL、BOOL、STRING 等
  • Scan Rate：扫描周期（毫秒）
  • Engineering Units：工程量单位（℃、bar、m³/h）
  • Low/High Raw：原始值范围
  • Low/High Eng：工程量范围
  • Alarm Limits：报警限值（Alarm High/Low、Warning High/Low）
模拟量转换示例

温度传感器：PLC 原始值 0-32767 对应 0-150℃

Raw Low = 0, Raw High = 32767, Eng Low = 0, Eng High = 150

CIMPLICITY 自动将原始值转换为工程量显示。

计算机与资源配置 工具

Computer（计算机）配置

Computer 是 CIMPLICITY 项目中的物理或虚拟计算机定义。每台计算机可以配置为 Server 或 Viewer 角色。

  • Server：运行项目配置、与 PLC 通信、管理数据
  • Viewer：运行 CimView 显示画面，连接到 Server 获取数据

Resource（资源）配置

Resource 是 CIMPLICITY 中的数据处理单元，每个 Resource 关联一个设备通信端口。点位必须分配到 Resource 才能采集数据。

最佳实践

为每个物理设备配置独立的 Resource，便于隔离报警和数据。例如：PLC1 的点位使用 Resource1，PLC2 的点位使用 Resource2。

在 Resource 配置中可以启用或禁用报警计数器（Alarm Counters）。

全局参数（Global Parameters）

CIMPLICITY 提供大量全局参数控制运行时行为，常用参数包括：

  • 自动登出时间（Auto Logout）
  • 安全通信（Use Secure Sockets）
  • Historian 连接配置
  • Server-Viewer 通信端口（默认 32000）
  • 画面文件自动分发路径

CimEdit 画面编辑器 工具

CimEdit 概述

CimEdit 是 CIMPLICITY 的画面开发工具，用于创建操作员可视化界面。开发完成的画面以 .cim 文件保存，在运行时通过 CimView 显示。

CimEdit 核心功能

  • 图形对象：矩形、圆、线、文本、位图、符号库对象
  • 点位绑定：将对象属性绑定到 Point ID，实现实时数据显示
  • 动画效果：颜色变化、位置移动、大小变化、可见性控制、旋转
  • 脚本编程：CimBasic 脚本，响应事件（点击、定时器、数据变化）
  • 画面导航：按钮跳转、弹出窗口、画面层级导航
  • .NET 组件嵌入：在画面中嵌入 .NET 控件

画面开发流程

  1. 在 Workbench 中打开 Screens 文件夹
  2. 新建或打开现有 CimEdit 画面（.cim 文件）
  3. 绘制图形对象（管道、设备图形、文本标签等）
  4. 配置对象属性 → Animation → 绑定 Point ID
  5. 配置动画（颜色、位置、大小、可见性）
  6. 添加交互按钮和导航脚本
  7. 保存画面 → 在 CimView 中测试
画面数量估算

项目开发前需估算画面数量：操作总览画面、过程详情画面、报警画面、趋势画面、配方画面、系统诊断画面等。

画面文件可从 Server 自动分发到各 Viewer 的共享文件夹。

🚨 报警与消息系统

报警配置、报警类别、报警路由、报警查看器、报警打印与OPC客户端

报警配置基础 概念

报警类型

CIMPLICITY 支持两大类报警：

  • 事件报警（Event Alarms）：基于系统事件或用户定义事件触发的报警，不直接关联单个点位。CIMPLICITY 自带一组系统事件报警。
  • 点位报警（Point Alarms）：基于点位值的报警，在 Point Properties 中配置。

系统事件报警（预置）

报警 ID功能说明
$ALARM_DISABLED指定点位的报警被禁用
$ALARM_ENABLED指定点位的报警被启用
$ALARM_MODIFIED指定点位的报警限值被修改
$DEVICE设备通信状态（up/down）
$DEVICE_DOWN指定设备通信故障
$DL_FILE_FULL日志文件已满
$LOGON / $LOGOUT用户登录/登出
$LOGIN_FAILURE用户登录失败
$DYN_CFG实体被动态修改
$AM_STATUS报警管理器状态（每半小时统计）
重要

系统事件报警可以被修改但绝不应被删除。它们是系统正常运行的基础。

点位报警与报警状态 概念

模拟量报警状态

模拟量点位支持最多 4 个报警值，每个对应一个报警状态：

点位值条件报警状态显示字符串
≥ Alarm HighAlarm HighAlarm High Msg
≥ Warning High 且 < Alarm HighWarning HighWarning High Msg
≤ Warning Low 且 > Alarm LowWarning LowWarning Low Msg
≤ Alarm LowAlarm LowAlarm Low Msg
< Warning High 且 > Warning LowNormalNormal Msg

数字量报警状态

数字量点位只有一个报警状态：值为 1 时报警（Alarm High Msg），值为 0 时正常或另一状态。

报警配置步骤

  1. 打开 Point Properties 对话框
  2. 切换到 Alarms 选项卡
  3. 配置报警限值（Alarm High/Low、Warning High/Low）
  4. 分配报警类别（Alarm Class）
  5. 配置报警字符串（Alarm String）
  6. 设置报警路由（Alarm Routing）
报警类别与路由 方法

报警类别（Alarm Classes）

报警类别用于将具有相似特征的报警分组管理。每个类别可以配置：

  • 颜色：报警在不同状态下的显示颜色（通过 rgb.dat 映射）
  • 优先级：报警优先级排序
  • 确认要求：是否需要操作员确认
  • 声音：报警触发时播放的声音文件
  • 日志记录：是否记录到日志和数据库

报警路由（Alarm Routing）

报警路由决定报警发送到哪些目标：

  • CimView Alarm Viewer（画面报警查看器）
  • Alarm Printer（报警打印机）
  • Alarm OPC Client（OPC 报警客户端）
  • Alarm Cast Server（报警推送服务器）
  • Historian（历史数据库）

报警字符串配置

报警字符串集（Alarm String Sets）通过索引号标识，为每个报警状态定义显示文本。支持多语言映射。

报警阻断（Alarm Blocking）

报警阻断配置一个层次结构，确保只接收最重要的报警。例如：上游设备故障时，抑制下游设备的连锁报警。

报警打印与 OPC 工具

报警打印机配置

  • 支持串口打印机、网络打印机和冗余打印机
  • 可配置打印格式：日期/时间格式、布局属性
  • 可设置全局打印机参数

报警 OPC 客户端

CIMPLICITY 可作为 OPC A&E（Alarms and Events）客户端，通过 OPC 服务器发送和接收报警数据。

报警管理器性能监控

System Sentry 提供 HMI Alarm Manager 计数器，用于监控系统报警性能：

  • Alarms Generated：累计产生的报警数（含重置后重新发送的）
  • Alarms Acknowledged：已确认的报警数
  • Alarms Reset：已复位的报警数
  • Alarm Count：当前活跃报警数
  • Update Queue Length：报警队列长度（应接近 0）
  • Alarms Generated/sec：每秒产生报警数

📈 趋势图配置

趋势控制配置、实时/历史趋势、运行时操作、数据分析功能

趋势图基础 概念

Trend Chart 概述

CIMPLICITY Trend Chart 是嵌入式趋势控件，可放置在 CimEdit 画面中，在运行时实时显示点位数据的历史趋势曲线。

趋势图显示特性

  • 多笔趋势：一个图表中显示多条趋势线
  • 实时模式：持续更新最新数据
  • 历史模式：查看指定时间段的历史数据
  • 缩放功能：放大特定区域数据
  • 游标分析：在趋势线上选择点查看精确数值和时间
  • 前后滚动：按行或按页前后滚动数据

配置步骤

  1. 在 CimEdit 中插入 Trend Control 对象
  2. 配置趋势笔（Pens）：每个 Pen 绑定一个 Point ID
  3. 设置 Pen 颜色、线宽、样式
  4. 配置图表属性：时间范围、Y 轴范围、网格
  5. 配置运行时操作选项
运行时操作 工具

运行时可配置属性

操作员在 CimView 运行时可以：

  • 更改趋势笔的点位（动态切换显示变量）
  • 修改图表时间范围和 Y 轴范围
  • 放大特定区域（Zoom）
  • 在选定时间点分析数据
  • 前后滚动数据（Scroll Lines Forward/Backward）
  • 更改颜色和字体
与 Historian 集成

Trend Chart 可连接到 Proficy Historian 获取长期历史数据，实现更长周期的趋势分析。在 Historian Connections 中配置连接。

📡 设备通信配置

端口与协议、设备配置、SRTP/Modbus/OPC通信、数据采集与诊断

通信架构与端口 概念

通信架构

CIMPLICITY 通过设备通信驱动与 PLC、控制器、仪表等设备通信。每个设备通过 Port（端口）和 Device（设备）配置。

配置层次

  1. Port（端口）：定义通信协议和物理连接参数
  2. Device（设备）：定义具体的 PLC/设备，关联到 Port
  3. Point（点位）：绑定到 Device，定义数据地址

端口配置步骤

  1. Workbench → Devices → Ports → 新建 Port
  2. 选择通信协议（SRTP、Modbus TCP、OPC 等）
  3. 配置端口参数（IP、端口号、超时等）
  4. 启用 Stale Data（过期数据）选项
Stale Data

启用 Stale Data On 时，当设备通信中断，点位保持最后有效值；关闭时，点位值变为无效。需根据应用需求选择。

通信协议详解 方法

协议用途典型设备

远程项目与冗余 方法

远程项目配置（Remote Project）

远程项目允许一个 CIMPLICITY Server 连接到另一个 Server 的项目，实现跨区域数据共享。

Server 冗余

冗余配置使用主备两台 Server，当主 Server 故障时自动切换到备用 Server。

  • Primary Server：正常运行时处理所有通信
  • Secondary Server：待机状态，监控 Primary 健康状态
  • Failover：Primary 故障时自动切换
  • Fallback：Primary 恢复后自动切回
冗余安全

冗余对的两台 Server 应部署在不同的物理位置或网络段，避免单点故障。需安全配置冗余对之间的通信。

  • 在 WebSpace Server 上安装 WebSpace 组件
  • 配置用户账号和权限
  • 通过浏览器访问：https://[server]:[port]/webspace

Terminal Services / RDP

也支持通过 RDP（远程桌面）连接到运行 CIMPLICITY 的计算机。

  • RDP Full Connection vs. RDP Web Connection
  • 建议修改默认 RDP 端口（3389）以增强安全

Basic Control Engine 概述 概念

Basic Control Engine

Basic Control Engine 是 CIMPLICITY 的脚本执行引擎，在 Server 端运行，负责处理事件、执行动作和脚本逻辑。

核心概念

  • Event（事件）：触发条件，如点位值变化、定时器到时、报警产生
  • Action（动作）：事件触发后执行的操作，如设置点位值、执行脚本、发送消息
  • Event Manager Resident Process (EMRP)：事件管理常驻进程，处理事件队列

多 Event Manager 实例

一个项目可配置多个 EMRP 实例，将不同的事件分配到不同的 EMRP，提高处理并行度和隔离性。

事件编辑器与动作配置 工具

事件编辑器（Event Editor）

事件编辑器用于创建和管理事件与动作的关联。

事件类型

事件类型触发条件
Point Event点位值满足指定条件
Timer Event定时器触发（一次性或周期性）
Alarm Event报警状态变化
System Event系统事件（如用户登录）

动作类型

  • Set Point：设置一个或多个点位值
  • Run Script：执行 CimBasic 脚本
  • Log Message：写入日志消息
  • Trigger Screen：在 Viewer 上弹出指定画面
  • Send Email/Message：发送通知

配置步骤

  1. 打开 Event Editor
  2. 创建新事件，选择事件类型
  3. 配置触发条件
  4. 创建动作，选择动作类型
  5. 将动作关联到事件
  6. 保存并更新项目

类与对象概念 概念

面向对象设计

CIMPLICITY 的 Classes and Objects 功能实现了面向对象的设计模式，类似于 PLC 编程中的 UDFB（用户自定义功能块）。

CIMPLICITY 概念PLC 概念说明
Class（类）Function Block Type模板，定义属性、数据项、脚本、画面
Object（对象）Function Block Instance类的实例，具体设备或控制对象
Attribute（属性）Input Parameter实例化时赋值的参数
Data Item（数据项）Point点位模板，实例化后生成实际点位
重要前提

配置类和对象前，必须先理解 CIMPLICITY 项目配置（点位、设备、报警等）。如果没有项目配置基础，类/对象配置会遇到困难。

类模板组成 方法

类模板包含的功能

功能说明
Attributes（属性）数值或字符串占位符，实例化时赋值。提供灵活性，确保不同对象有不同的值
Data Items（数据项）点位模板，使用属性定义。实例化时生成实际点位
Scripts/Actions/Events脚本、动作和事件，可引用数据项。适用于类中所有对象
CimEdit Class Screens类画面，包含图形对象。可选默认画面
Composite Members组合成员，引用其他类。可有自己的数据项、事件和动作
Faceplate面板画面，左键点击对象时弹出

类版本管理

  • 类版本号在结构变更时自动递增（添加/删除数据项、组合成员）
  • 可手动分配版本号
  • 导入低版本类到高版本时会警告

属性配置

属性可以指定在实例化时的处理方式：

  • String Substitution：使用 $OBJECT 等变量进行字符串替换
  • Device ID：每个对象支持一个设备，在 Object 级别分配
  • Address：地址在 Data Item 或 Object 级别配置，优先级：Data Item > Object
对象实例化过程 工具

实例化三步流程

Class → Object → Point 实例化流程
A. Class: Data Item 配置（定义模板）
  → 属性占位符、设备类型、地址格式
B. Object: 创建对象（赋值属性）
  → 对象名、Device ID、属性值、地址
C. Instantiation: 生成实际点位
  → {$OBJECT}. 完整点位 ID
实例化示例

Class: TANKS

Data Item: VLVBP (阀门控制点位)

String Substitution: $OBJECT = STATION01

Device ID: TRIPLEXDEV

Address: %5R

结果：生成点位 STATION01.VLVBP，设备 TRIPLEXDEV，地址 %5R

对象在 Workbench 中的管理

  • 类在 Workbench 右侧窗格列表显示
  • 类可从项目中导出或导入到其他项目
  • 对象通过 Object dialog box 创建和管理

网络分区与信任等级 概念

Purdue 网络分区模型

CIMPLICITY 安全部署遵循分层信任模型：

区域信任等级说明
Control System Zone最高CIMPLICITY Server、PLC、控制器
Control System DMZ中高Historian、Alarm Cast Server
Support DMZ中低更新服务器、监控
Corporate Zone最低企业网络
安全原则
  1. 低信任区域的连接应被限制，通过白名单管理
  2. 高信任区域应主动发起与低信任区域的通信
  3. 推荐使用 OPC UA 实现端点双向认证和数据加密
  4. 使用 IPSec 加密控制系统区域内通信
  5. Use Secure Sockets 选项提供 RC4 54位加密（基础级别）

  • OPC UA Server 配置
  • Change Management（变更管理）

Configuration Hub 安全

Configuration Hub 作为反向代理服务器，与其他 Proficy 产品的 Web 服务器通过 HTTPS 通信。默认不验证 SSL 证书，建议配置 SSL 验证以防止中间人攻击。

SSL 验证配置
  1. 收集所有 Web 服务器的根证书（uaa_root_cert.crt、InteropRootCA.crt、CimScadaConfigRootCA.crt）
  2. 合并为 bundle.crt 文件
  3. 在 Configuration Hub 的 Apache 配置中设置：SSLProxyVerify require、SSLProxyCheckPeerCN on、SSLProxyCheckPeerName on

System Sentry 与数据库日志 方法

System Sentry

System Sentry 是 CIMPLICITY 的系统性能监控工具，提供：

  • HMI Alarm Manager 计数器：监控报警系统性能
  • 进程状态：CIMPLICITY 各服务进程运行状态
  • 资源使用：CPU、内存、网络使用率
  • 队列监控：Update Queue、Application Queue、PTMRP Queue 长度

Database Logger

Database Logger 将 CIMPLICITY 数据记录到外部数据库：

  • 支持数据库：Microsoft Access、SQL Server、Oracle
  • 日志类型：点位数据日志、组点位日志、报警日志、事件报警日志、事件管理日志、状态日志、应用日志
  • 动态配置：运行时修改日志条件无需更新项目
  • 表对齐（Reconcile）：同步 CIMPLICITY 表和数据库表
Historian 集成

对于长期数据存储和分析，建议使用 Proficy Historian 而非 Database Logger：

  1. 安装 Historian 时选择 Archive Features
  2. 启用 Historian OPC Interface
  3. 选择需要记录的点位
  4. 配置 Historian 连接（在 CIMPLICITY Options 中）

  • Alarm Cast：报警通知和升级系统
  • WebSpace：Web 远程访问
  • Change Management：版本控制
