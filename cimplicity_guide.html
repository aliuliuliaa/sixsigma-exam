第 1 章

系统概述与架构

CIMPLICITY HMI/SCADA 系统架构、核心组件、数据采集原理与部署模式
CIMPLICITY 系统总览 概念

CIMPLICITY 是 GE Vernova（原 GE Digital）开发的企业级 HMI/SCADA 监控系统，广泛应用于离散制造、过程控制、基础设施监控等领域。它提供从数据采集、实时监控、报警管理到历史趋势分析的完整功能链。

组件功能典型部署

CimView (Viewer)操作员界面，显示画面、报警、趋势操作站 / 工程师站
Workbench工程开发环境，配置项目、点位、画面工程师工作站
WebSpaceWeb 远程访问，通过浏览器查看画面DMZ 区域服务器
Historian长期历史数据存储与分析独立数据库服务器
数据采集原理 概念

CIMPLICITY 通过轮询机制从 PLC 和其他现场设备读取数据。数据流路径为：

关键概念

概念说明示例
Point（点位）数据的最小单元，对应 PLC 中的一个寄存器或标签TIC_101_PV（温度值）
Device（设备）一个通信连接，对应一台 PLC 或网关PLC1（IP: 192.168.1.10）
Port（端口）通信端口配置，定义协议和连接参数Port 0: SRTP, 18245/TCP
Block（块）一组连续寄存器的批量读取，优化通信效率Block 1: %R100-%R199
性能优化合理配置 Block 大小可显著减少通信负载。建议将相邻地址的点位归入同一 Block，Block 大小建议 100-500 个寄存器。
部署模式 方法

三种典型部署模式

模式架构适用场景优缺点
单机模式Server + Viewer 在同一台机器小型系统、测试开发成本低；无冗余
客户/服务器1台 Server + 多台 Viewer中型工厂、多操作站集中管理；Server 故障则全停

项目创建流程 方法

新建项目步骤

  1. 创建项目：File → New → Project，指定项目名称和存储路径
  2. 配置角色：设置项目角色（Primary/Secondary），定义项目部署架构
  3. 创建设备：在 Device Communication 中添加 PLC 设备，配置协议和 IP
  4. 创建端口：定义通信端口参数（端口号、协议、超时）
  5. 添加点位：在 Point Table 中定义数据点，关联设备地址
  6. 开发画面：在 CimEdit 中创建操作界面
  7. 配置报警：设置报警阈值和通知规则
  8. 编译部署：编译项目并部署到运行时服务器

画面开发与 CimEdit

CimEdit 画面编辑器、11 种动画类型、对象属性配置与画面开发最佳实践
CimEdit 画面开发概览 工具

CimEdit 是 CIMPLICITY 的所见即所得画面编辑器，支持丰富的图形对象、动画效果和脚本编程。画面文件扩展名为 .cim，运行时由 CimView 加载显示。

11 种动画类型详解 概念

CimEdit 提供 11 种核心动画类型，每种动画绑定一个或多个点位，实现图形对象的动态变化。

动画类型功能说明绑定属性典型应用

Color（颜色变化）根据点位值范围改变对象颜色阈值/颜色映射表温度高低变色、状态红绿
Position（位置移动）根据点位值改变对象 X/Y 坐标Min/Max → 像素范围传送带移动、阀门开度
Rotation（旋转）根据点位值旋转对象角度范围映射电机转动、仪表指针
Size（尺寸变化）根据点位值改变对象宽高Min/Max 尺寸液位高度、容器填充
Text（文本显示）显示点位值或映射文本值→文本映射表状态文字、数值显示
Fill（填充比例）按百分比填充对象内部区域0-100% 映射储罐液位条形图

SmartObjects（智能对象）组合多个动画的复合对象多点绑定电机综合显示组件
最佳实践组合使用 Fill + Color 动画可实现最直观的液位显示：液位高度用 Fill 表示，超过阈值自动变色用 Color 实现。

报警与消息系统

报警状态机、报警分类、Alarm Cast 分级、通知规则与报警管理最佳实践
报警状态机 概念

CIMPLICITY 报警系统基于状态机模型，每个报警点可以处于以下状态之一。状态转换由过程变量（PV）与预设阈值的比较结果触发。

报警状态定义

状态触发条件颜色操作员动作
NORMALPV 在正常范围内绿色无需操作
WARNING_HIGHPV > 高警告阈值黄色关注，准备干预
ALARM_HIGHPV > 高报警阈值红色立即处理
WARNING_LOWPV < 低警告阈值黄色关注，准备干预
ALARM_LOWPV < 低报警阈值红色立即处理
Alarm Cast 报警分级 方法

Alarm Cast 是 CIMPLICITY 的报警分发和升级机制，支持将报警按优先级路由到不同的接收者和显示设备。

优先级级别显示方式通知方式
P1 - 紧急Critical全屏弹窗 + 闪烁短信 + 邮件 + 声光报警
P2 - 高High报警栏置顶 + 颜色标记邮件 + 声音提示
P3 - 中Medium报警栏列表声音提示
P4 - 低Low仅记录日志无主动通知
设计原则报警分级应遵循"少而精"原则 — P1 报警不应超过总报警量的 5%，避免报警泛滥导致操作员麻木。
报警管理最佳实践 方法

EEMUA 191 报警管理标准

指标推荐值说明
平均报警率< 1 条/分钟正常运行时
峰值报警率< 10 条/分钟扰动期间
陈旧报警< 5 个/操作站存在超过 30 天的报警
报警泛滥持续时间< 10 分钟连续超过 10 条/分钟的时段
常见问题报警泛滥是工业系统中最严重的问题之一。定期审查并优化报警阈值，删除无效报警，合理配置延迟时间，是保持报警系统有效性的关键。

Basic Control Engine 概述 概念

Basic Control Engine (BCE) 是 CIMPLICITY 的内嵌脚本引擎，基于 CimBasic 语言（类 VBScript 语法），支持事件驱动的自动化逻辑。

BCE 脚本类型

类型触发方式典型用途
Point Script点位值变化时触发数据转换、联动控制
Screen Script画面事件（打开/关闭/定时）画面初始化、定时刷新
Object Script对象事件（点击/悬停）按钮操作、弹窗触发

CimBasic 脚本语法 工具

常用 CimBasic 代码示例

读取和写入点位 
' 读取点位值
Dim tempValue
tempValue = PointGet("TIC_101_PV")

' 写入点位值
PointSet("TIC_101_SP", 75.0)

' 读取点位状态
Dim pointState
pointState = PointGet("TIC_101_PV.STATE")
条件判断与报警联动 
Sub PointChange()
    Dim pv, sp
    pv = PointGet("TIC_101_PV")
    sp = PointGet("TIC_101_SP")
    
    If Abs(pv - sp) > 5.0 Then
        ' 偏差过大，触发联锁
        PointSet("INTERLOCK_ALARM", 1)
        LogMessage "温度偏差超限: PV=" & pv & " SP=" & sp
    ElseIf Abs(pv - sp) < 1.0 Then
        ' 偏差恢复正常
        PointSet("INTERLOCK_ALARM", 0)
    End If
End Sub
调试技巧使用 LogMessage 将调试信息写入系统日志，在 Workbench 的 Event Log 窗口中查看。避免在 BCE 脚本中使用 MsgBox，这会阻塞脚本引擎。

PID 回路控制集成 概念

CIMPLICITY 可与 GE PACSystems PLC 的 PID 回路集成，实现监督控制和设定值管理。

PID 控制架构

层级组件功能

PID 模式与操作

模式说明操作员动作
AUTO（自动）PLC 执行 PID 运算，自动调节 CV仅修改 SP
MANUAL（手动）操作员直接设定 CV 输出修改 CV
CASCADE（串级）主回路 SP 由上级回路输出提供修改主回路 SP
安全切换从 MANUAL 切换到 AUTO 前，必须确认 SP 接近当前 PV，避免输出突变。建议配置无扰动切换 (Bumpless Transfer) 功能。

变更管理流程 方法

标准变更管理流程 (ITIL for OT)

步骤内容责任人
1. 变更申请提交变更请求 (RFC)，描述变更内容和影响申请人
2. 影响评估评估变更对生产、安全、合规的影响工程师 + 安全负责人
3. 审批根据影响等级获得相应审批CAB (变更顾问委员会)
4. 测试在测试环境验证变更工程师
5. 实施在维护窗口执行变更工程师 + 操作员
6. 验证确认变更后系统正常运行操作员 + 工程师
7. 归档记录变更内容、版本号、回滚方案工程师
回滚计划每次变更都必须准备回滚方案。记录变更前的版本号和备份位置，确保在变更失败时能在 15 分钟内恢复。
