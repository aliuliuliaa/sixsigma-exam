🏗️一、程序框架与项目结构

PMEPME 工程结构

Project (工程, 最多32个Target) ├─ Target 1: PAC控制器 (RX3i CPU) │ ├─ Hardware Configuration (硬件组态 HWC) │ │ ├─ CPU模块 (CPE305/310/330/400) │ │ ├─ 电源模块 │ │ ├─ I/O模块 (DI/DO/AI/AO/HSC) │ │ ├─ 通信模块 (ETM001/PNC001) │ │ └─ 扩展机架/远程IO站 │ ├─ Logic (程序逻辑) │ │ ├─ _MAIN (主程序块, 自动执行) │ │ ├─ Program Block (程序块) │ │ ├─ Parameter Block (参数块) │ │ ├─ UDFB (用户自定义功能块) │ │ └─ External Block (外部C块) │ ├─ EGD (以太网全局数据配置) │ ├─ Motion / PACMotion (运动控制) │ ├─ VersaSafe (安全控制) │ └─ DLB (诊断逻辑块) ├─ Target 2: 触摸屏 (View) │ └─ 画面组态 └─ Variables (共享变量表)

类型编程语言最大大小参数说明
程序块LD/ST/FBD128KB0入/1出(OK)_MAIN自动执行，其它需被调用

程序执行流程

上电 → 读取硬件配置 → 初始化I/O ↓ ┌─→ 输入采样阶段: 读取所有%I和%AI到映象区 │ ↓ │ 用户程序执行: _MAIN → 调用各子程序块 │ ↓ │ 输出刷新阶段: 将%Q和%AQ写到输出模块 │ ↓ │ 通信处理: SRTP/EGD/Modbus窗口 │ ↓ └─── 循环 (扫描周期) 扫描模式: - Normal: 正常扫描，尽可能快 - Constant Window: 固定窗口时间，剩余时间给通信 - Constant Sweep: 固定扫描周期，超时报警(Watchdog)
建议：对控制精度要求高的场合用Constant Sweep模式，设定固定扫描周期(如10ms/20ms)。

项目编程规范建议

  • 地址规划：做Excel文件规划各模块的%I/%Q/%AI/%AQ地址，以及状态/诊断地址
  • 模块化编程：_MAIN只做调用，逻辑写在子程序块中
  • UDFB复用：电机控制、阀门控制等重复逻辑封装成UDFB
  • 变量命名：用有意义的符号变量名，如Motor1_Start、Tank_Level_PV
  • 状态诊断：每个I/O模块分配状态地址(建议放高地址区如%R9000+)
  • 注释：每个块和关键梯级加Comment注释
  • 备份：每次修改后用Back Up功能打包zip

符号变量 vs 映射变量

映射变量：直接使用物理地址，如 %R0001、%I00001

符号变量：给地址取名，如 Motor_Start 映射到 %I00001

// 在Navigator的Var.标签页声明变量 // Name Type Address Retentive Description // Motor_Start BOOL %I00001 No 电机启动按钮 // Motor_Run BOOL %Q00001 No 电机运行输出 // Tank_Level INT %AI0001 No 水箱液位 // Set_Pressure REAL %R00001 Yes 设定压力值
推荐使用符号变量编程，可读性好，修改硬件时不用改程序。Enable Shared Variables属性可让多个Target共享变量。

间接寻址

PACSystems CPU和90-70系列支持间接寻址，可用于所有寄存器类型(%R, %P, %L, %W, %AI, %AQ)：

// 用@符号进行间接寻址 // %R0001 := @%R0100; // 读取%R0100中存储的地址所指向的值 // // 实际应用：遍历数组 // FOR index := 1 TO 100 DO // @Register_Pointer := count; // Register_Pointer := Register_Pointer + 1; // END_FOR;

内存配置（CPU Memory卡）

在CPU的Memory参数卡中可配置各区域大小：

  • Reference Points：各类位地址空间(%I/%Q/%M/%T等)，只读
  • Reference Words：各类字地址空间(%AI/%AQ/%R等)，可配置
  • Managed Memory：符号变量地址空间(%W)，可配置
  • Total User Memory Required：用户数据内存需求总计
默认内存值可能太小，使用EGD或大量符号变量时需要手动调大。下载硬件配置后生效。
🧩三、程序块与UDFB（面向对象编程）

UDFB用户自定义功能块（类似Class）

UDFB是PACSystems的面向对象编程核心。定义好UDFB后可创建多个实例，每个实例有独立的内部数据拷贝。

// UDFB结构 UDFB: Motor_Control ├─ 输入参数 (只读): │ ├─ Start (BOOL) - 启动命令 │ ├─ Stop (BOOL) - 停止命令 │ ├─ EStop (BOOL) - 紧急停止 │ ├─ Motor_Ready(BOOL) - 电机备妥信号 │ └─ Motor_FB (BOOL) - 运行反馈 ├─ 输出参数 (只读): │ ├─ Motor_Run (BOOL) - 运行输出 │ ├─ Alarm (BOOL) - 报警输出 │ └─ OK (BOOL) - 块状态(Y0,自动) ├─ 成员变量 (实例数据): │ ├─ Run_Timer (DINT) - 运行计时器 │ ├─ Start_Count(INT) - 启动次数 │ └─ Fault_Code (WORD) - 故障代码 └─ 逻辑 (LD/ST/FBD) // 调用时指定实例名: // CALL Motor_Control (实例: Conveyor_1) // CALL Motor_Control (实例: Conveyor_2) // 每个实例独立运行，数据互不影响

UDFB三种参数传递机制

机制方向说明使用场景
参考变量传递双向传递地址，直接操作原变量默认机制，最常用
值传递入→内拷贝值到UDFB，不影响原变量需要保护原数据时
结果传递内→出拷贝值进入，退出时拷贝结果回原变量输出参数
BOOL输入默认参考变量传递，输出默认结果传递(Y0除外)。形参不能用于变位线圈/触点、DO I/O功能块、间接寻址。

UDFB实际项目示例：带启动预警的电机控制

需求

  • 启动命令发出后等待10秒再启动电机
  • 10秒等待中产生灯光闪烁(1秒脉冲)和电铃报警
  • 可用于多台电机，每台独立控制

UDFB定义: UDFB_LDBK

输入参数: Start_PB (BOOL) - 启动按钮 Stop_PB (BOOL) - 停止按钮 Ready (BOOL) - 备妥信号 输出参数: Motor_Out (BOOL) - 电机输出 Horn (BOOL) - 电铃输出 Light (BOOL) - 灯光输出 成员变量: Timer_Val (DINT) - 延时计时 逻辑(LD): Start_PB AND NOT Stop_PB AND Ready → 启动内部继电器 启动继电器 AND NOT Motor_Out → 启动延时定时器(10s) 定时器运行中: Light := %S0005 (1秒脉冲闪烁) Horn := TRUE (持续电铃) 定时器到达 → Motor_Out := TRUE Motor_Out AND Stop_PB → 复位 调用(_MAIN中): CALL UDFB_LDBK (实例: Conveyor_1) Start_PB := C1_Start_PB Stop_PB := C1_Stop_PB Ready := C1_Ready Motor_Out → C1_Motor Horn → C1_Horn Light → C1_Light CALL UDFB_LDBK (实例: Conveyor_2) Start_PB := C2_Start_PB ...

参数块示例：电机启停控制

// 参数块 PB_LDBK // 输入: Start, Stop, EStop, MR(备妥), R_M(运行反馈) // 输出: Motor_Run // LD逻辑: // Start AND NOT Stop AND NOT EStop AND MR → 内部启动继电器 // (启动继电器 OR R_M) AND NOT Stop AND MR → Motor_Run // Motor_Run 保持依赖 R_M 反馈 // _MAIN中调用: // CALL PB_LDBK(Start=M1_Start, Stop=M1_Stop, EStop=M1_ES, // MR=M1_Ready, R_M=M1_Return, Motor_Run=M1_Output)

程序块调用关系

_MAIN (自动执行) ├─ CALL PB_LDBK (参数块: 电机控制) ├─ CALL UDFB_LDBK (实例: Conv_1) ← UDFB调用 ├─ CALL UDFB_LDBK (实例: Conv_2) ← 同一UDFB不同实例 ├─ CALL Valve_Ctrl (参数块: 阀门控制) ├─ CALL Comm_Handler (程序块: 通信处理) ├─ CALL Alarm_Handler (程序块: 报警处理) └─ CALL PID_Loop (参数块: PID控制) 注意: - 不要从_MAIN或非中断块中调用中断驱动块 - 嵌套调用层数受堆栈限制 - 子程序可以调用其它子程序(包括自己=递归)

📝五、结构化文本(ST)编程

ST语言概述

ST是类似PASCAL的高级文本语言，适合复杂数学计算、算法和决策。支持符号变量，类似句子，易于理解。

ST基本语法

// 赋值 Motor := Switch_1 AND Switch_2; Temperature := (Raw_Value - 0) * 100.0 / 32000.0; // 调用程序块 My_LDBlock(); // 调用带参数的函数(实参顺序) abs_int(myInput, myOutput); // 调用带形参的函数(顺序无关) abs_int(IN := myInput, Q => myOutput, ENO => mySuccess); // 注释 (* 这是块注释 *) // 这是行注释

条件分支

// IF-THEN-ELSIF-ELSE-END_IF IF (myDint = 0) THEN myTag := 1; ELSIF (myDint2 > 0) THEN myTag := 2; ELSE myTag := 3; END_IF; // CASE语句 CASE Proc_State OF INIT_STATE: status := INITIALIZED; FILL_STATE: status := FILLING; EMPTY_STATE: status := EMPTYING; ELSE status := UNKNOWN; END_CASE;

循环语句

// FOR循环 FOR count := 1 TO 100 BY 2 DO @Register_Pointer := count; Register_Pointer := Register_Pointer + 1; END_FOR; // WHILE循环 WHILE INT_index < 100 DO INT_temp := @INT_index; INT_index := INT_index + 1; @INT_index := INT_temp; END_WHILE; // REPEAT循环 REPEAT INT_temp := @INT_index; INT_index := INT_index + 1; @INT_index := INT_temp; UNTIL INT_index = 100 END_REPEAT; // EXIT: 退出循环(用在WHILE和REPEAT中) // RETURN: 退出程序块

EGDEGD以太网全局数据（详细配置）

EGD使一个CPU(Producer)周期性分享内部数据给其他CPU(Consumer)。

EGD交换(Exchange)的三个标识: Producer ID - 生产者IP地址 Exchange ID - 交换包标识号 Adapter Name - 以太网接口标识号 配置注意事项: 1. Local Producer ID可以不是实际IP，只是唯一标识 2. Consumer的数据类型和长度必须与Producer一一对应 3. Unicast一对一, Multicast一对多(广播) 4. Consumer的update timeout设为Producer周期的3倍以上 5. Producer周期小于100ms时会占用大量带宽，影响PME/HMI连接 6. SPEC: 最多32个group，一个exchange最多1400 bytes

EGD配置实例

PAC_1 (192.168.101.11) → PAC_2 (192.168.101.12) PAC_1的%I1~I256, %R1~R20 → PAC_2的%M1~M256, %AI1~AI20 PAC_1 (Producer)配置: Ethernet Global Data ID: 10.10.10.101 Adapter Name: 机架0槽位1的ETM001 Exchange ID: 1 Destination(Consumer Group): 10.10.10 Produced Period: 50ms 数据: %I(256点) + %R(20字) PAC_2 (Consumer)配置: Consumed Exchange: Producer ID: 10.10.10.101 Exchange ID: 1 Update Timeout: 200ms (约3倍Producer周期) 接收数据存储到: %M(256点) + %AI(20字) 注意: Consumer侧接收地址不能被实际I/O模块占用!

🌡️八、PID控制

PIDPID功能块

PACSystems提供两种PID算法：

  • PID_ISA: 标准ISA PID算法(Interacting)
  • PID_IND: 独立项算法(Non-interacting)

PID控制块结构(40个字)

PID参数块: 40个字(前13个可配置, 后27个CPU使用) 字0: SP(设定值) - 与PV同范围 字1: PV(过程变量) - 来自传感器 字2: 采样时间(ms) - 最小10ms 字3: KP(比例增益) 字4: KI(积分增益) 字5: KD(微分增益) 字6: CV偏差(死区) 字7: PV下限 字8: PV上限 字9: CV下限 字10: CV上限 字11: CV最小变化时间(ms) 字12: 手动命令值(手动模式时设CV) 字13: 控制输出CV(自动模式输出) 字14~39: CPU内部使用(不可配置)

PID操作模式

自动模式: Enable=ON, Manual=OFF → PID自动计算CV输出 → CV存储在%Ref+13 → 手动命令跟踪CV值(无冲击切换) 手动模式: Enable=ON, Manual=ON → CV = 手动命令值(%Ref+12) → UP/DN输入可手动增减CV → 积分器持续更新(无冲击切换) 关键特性: - 最小执行间隔: 10ms - 最大执行间隔: 10.9分钟 - 所有参数为16位整型 - PV/SP缩放范围: 0~32000 或 0~10000(0.00%~100.00%) - 一个PID函数不能在一个扫描周期内多次调用 - 每个PID必须用不同的40字存储区域

PID参数整定建议

  • 先设KI=0, KD=0，只调KP到系统振荡
  • KP设为振荡值的60%
  • 逐步增加KI直到系统再次振荡，设KI为振荡值的50%
  • 如有需要逐步增加KD减少超调
  • 使用临时线圈监控PID参数变化
  • 调好后可将参数作为常量用BLKMOV定义
长时间延时后调用自动模式PID，需用SVC_REQ 16或51更新%Ref+23(最后运行时间)，避免积分器出错。
⚡九、中断驱动块

三种中断类型

类型触发源说明
时间中断CPU定时器按指定时间间隔执行，可设初始延时

时间中断配置

在块的Scheduling属性中配置: Time Base(时基): 1.0s / 0.10s / 0.01s / 0.001s Interval(间隔): 时基倍数 Delay(延时): 可选, 首次执行的额外延时 首次执行时间 = (延时×时基) + (间隔×时基) 示例: 1ms定时中断 Time Base: 0.001s Interval: 1 → 每1ms执行一次

中断调度模式

模式中断数优先级说明
普通块调度64个IO/模块中断 + 16个时间中断相同默认模式
优先级块调度32个中断触发可分配高优先级可打断低优先级
不要从_MAIN或非中断块中调用中断驱动块！中断块使用独立堆栈，嵌套调用过多会导致"应用堆栈溢出"故障。

  • 仔细看Feedback Zone的内容和Fault Table的内容
  • Error可能是同一工程下其他Target的编译错误
  • 可能是SYS FLT没有清空
