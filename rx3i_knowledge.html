🖥️一、硬件架构

PACSystems 系列定位

RX3i 属于 Emerson（原 GE Fanuc）PACSystems 家族，是 PAC（可编程自动化控制器），不是传统 PLC。融合了 PLC 的确定性控制 + DCS 的系统级管理 + 计算机的开放性。

  • RX7i：高端机架式，VME 总线，大型过程控制
  • RX3i：中端机架式，PCI 总线，中小型应用最常用
  • RSTi-EP：紧凑型，IP20 导轨安装

RX3i 背板系统

背板类型总线槽数说明
通用背板PCI（双总线）10/16槽支持 IC695* 和 IC694* 模块混插
串行扩展背板串行总线最多16槽通过扩展电缆连接，距离最远100m
通用背板有两条独立总线：PCI 总线（高速，给 IC695 模块用）和串行总线（给 IC694 模块用），可在同一背板上混用两代模块。

电源模块

  • IC695PSD040：120/240VAC，40W
  • IC695PSD140：120/240VAC，40W（增强版）
  • IC695PSA040：24VDC，40W
  • IC695PSA140：24VDC，40W（增强版，支持更多背板负载）
电源功率必须 ≥ 背板上所有模块功耗之和，建议预留 20% 余量。

CPU 内置接口

  • 以太网口（CPE310/330/400）：支持 SRTP、Modbus TCP、EGD、OPC UA
  • 串口（RS-232）：支持 SNP、Modbus RTU
  • USB 口：编程调试用（不支持运行时连接）
  • PROFINET：需配 PNC001 模块实现

CPU 工作模式

模式说明
STOP停止执行用户程序，I/O 不更新
RUN正常执行用户程序，I/O 正常更新
RUN-IO-Disabled执行程序但 I/O 不输出（调试用）
STOP-IO-Enabled程序不执行但 I/O 保持输出（维护用）
模式切换可通过 PME 软件、CPU 面板开关或系统变量 %S 实现。

I/O 寻址方式

RX3i 支持两种寻址方式：

  • 机架-槽位寻址：%I00001 对应机架0、槽位1的输入点0
  • 别名引用：在 PME 中给地址取变量名，如 Start_Button 映射到 %I00001
%I 机架(1位) 槽位(2位) 点号(2位)
%I 0 01 01 = 机架0 槽位1 点01
%AI 机架(1位) 槽位(2位) 通道(2位)
推荐使用别名引用，程序可读性更好，修改硬件时不用改程序。

I/O 诊断功能

  • 模块级：电源故障、接线故障、短路/断路检测
  • 通道级：每个通道独立的诊断状态字
  • 热插拔：IC695 系列支持带电更换模块
  • 故障状态映射到 %SA/%SB 系统变量
💻四、PME 编程软件

PME软件概述

PAC Machine Edition (PME)（原 Proficy Machine Edition）是 Emerson 的统一工程环境，集成了：

  • Logic Developer - PLC：PLC 编程（LD/ST/FBD/IL/SFC）
  • Logic Developer - Motion：运动控制编程
  • View Developer：操作面板画面开发
  • Navigator：项目管理导航器

五种编程语言

语言全称适用场景
LDLadder Diagram 梯形图逻辑控制、继电器逻辑（最常用）
STStructured Text 结构化文本数学运算、数据处理、复杂算法
FBDFunction Block Diagram 功能块图过程控制、闭环控制
ILInstruction List 指令表类似汇编（使用较少）
SFCSequential Function Chart 顺序功能图流程控制、状态机
实际工程中最常用的是 LD + ST 混合编程：主逻辑用 LD，数据处理和通信用 ST。

定时器与计数器

定时器（%T）：

  • 基准时间：1ms / 10ms / 100ms / 1s
  • 当前值在 %Tnnnn 寄存器
  • 状态位在 %Tnnnn 对应的离散位
TMR T1: 100ms * 50 → 5秒定时
// %T1.Current = 0~50
// %T1.Status = 1 当计时到达

计数器（%C... 不，RX3i 用通用寄存器实现）：

RX3i 没有专用计数器指令，用 MATH + 比较指令实现：

%R10 = %R10 + 1 // 每次触发加1
// 比较判断：
// CMP %R10 >= 10 → SET Coils

SRTP 通信配置

SRTP（Service Request Transport Protocol）是 GE/Emerson 私有协议，用于：

  • PME 编程软件 ↔ CPU（上传/下载/调试）
  • CPU ↔ CPU 之间的数据交换
  • CPU ↔ Cimplicity 上位机数据采集
// 配置步骤：
// 1. PME → Hardware Config → CPU → Ethernet Port
// 2. 设置 IP 地址、子网掩码、网关
// 3. 设置 PLC Name（Cimplicity 连接用）
// 4. 下载硬件配置到 CPU

// 连接数限制：
// CPE310: 最多 16 个 SRTP 连接
// CPE330: 最多 32 个 SRTP 连接

EGD 广播通信

EGD（Ethernet Global Data）用于多台 PLC 之间的高速数据共享，基于 UDP 广播：

  • 一台 PLC 发送（Producer），多台接收（Consumer）
  • 周期性发送，典型 1ms~100ms
  • 数据存储在 %G 全局数据区
  • 适合：多 PLC 同步、状态广播

Modbus TCP 通信

// CPU 作为 Modbus TCP Server (从站)
// 端口: 502
// 地址映射:
// %I → Coil (0x)
// %Q → Coil (0x) / Holding Reg (4x)
// %AI → Input Reg (3x) / Holding Reg (4x)
// %R → Holding Register (4x)

// CPU 作为 Modbus TCP Client (主站)
// 使用 CHAN_FUNC 指令主动读写

OPC UA Server

CPE310/330/400 内置 OPC UA Server，可直接与上位机通信：

  • 端口：4840（默认）
  • 安全模式：None / Sign / Sign&Encrypt
  • 认证：匿名 / 用户名密码 / 证书
  • 地址空间：变量直接映射为 OPC UA 节点
OPC UA 是未来趋势，推荐新项目优先使用 OPC UA 替代 SRTP 连接上位机。
📊六、Cimplicity 上位机

SCADACimplicity 概述

CIMPLICITY 是 Emerson 的 SCADA/HMI 上位机软件，用于：

  • 过程监控和数据采集
  • 报警和事件管理
  • 历史数据记录和趋势分析
  • 操作员界面（HMI）
  • 报表生成

与 PLC 的连接方式

方式协议说明
GE SRTPSRTP (TCP 18245)最常用，直连 RX3i CPU
OPC UAOPC UA (TCP 4840)标准化接口，跨平台
OPC DAOPC DA (DCOM)老方式，需 OPC Server 中间件
Modbus TCPModbus (TCP 502)简单数据采集
推荐：新项目用 OPC UA；老项目维护用 SRTP；与第三方设备用 Modbus TCP。

趋势与历史数据

  • 实时趋势：显示最近 N 分钟数据，自动滚动
  • 历史趋势：查询指定时间段数据，支持缩放
  • 历史数据库：默认使用 Cimplicity 自带历史库，也可配置 SQL Server
  • 采样周期：最小 1 秒，可配多级归档（秒→分钟→小时）

固件升级

  • 通过 PME → Tools → Firmware Update
  • 支持 CPU、通信模块、I/O 模块固件升级
  • 升级前备份项目和硬件配置
  • CPU 固件升级需在 STOP 模式下进行

  • 项目备份：PME → File → Save As / Export
