1.1 六西格玛的起源与发展 概念

起源

六西格玛由摩托罗拉公司工程师比尔·史密斯（Bill Smith）于1986年提出。核心目标是将缺陷率降至每百万次机会中不超过3.4个（3.4 DPMO）。

发展历程

时间里程碑关键人物/公司
1986年六西格玛概念诞生比尔·史密斯/摩托罗拉
1988年摩托罗拉获首届鲍德里奇奖摩托罗拉
1995年GE全面推行六西格玛杰克·韦尔奇/GE
2000年后六西格玛与精益融合全球推广

三个层次含义

  • 统计层面：过程均值距规格限6个标准差，缺陷率3.4 DPMO，Cpk>=2.0
  • 方法论层面：DMAIC改进模型 + DFSS设计模型
  • 管理系统层面：以客户为中心、数据驱动、持续改进的组织文化
六西格玛数学本质
当过程均值偏移1.5sigma时：P(缺陷) = 2 x Phi(-4.5) = 3.4 x 10^-6
记忆口诀摩托罗拉86年提出，3.4 DPMO是标志；韦尔奇95年推广，GE案例最经典。

1.3 精益生产与六西格玛 概念

精益生产

起源于丰田生产方式(TPS)，核心是消除浪费，通过JIT和自働化实现高效运营。

精益五大原则（V-V-F-P-P）

  • 价值(Value)：从客户角度定义价值
  • 价值流(Value Stream)：识别创造价值的全流程
  • 流动(Flow)：让价值连续流动
  • 拉动(Pull)：按客户需求拉动生产
  • 完善(Perfection)：持续追求完美

精益 vs 六西格玛对比

维度精益六西格玛
焦点速度和效率(消除浪费)质量和一致性(减少变异)
核心方法VSM、5S、看板、SMEDDMAIC、DOE、SPC、假设检验
问题类型流程慢、库存多变异大、缺陷多
工具侧重定性分析定量统计分析

精益六西格玛(LSS)：精益解决"快"，六西格玛解决"好"，两者融合追求持续改进。

考试要点精益五大原则(V-V-F-P-P)常出多选题。七大浪费(见第七章)是高频考点。

1.6 质量成本(COQ) 方法
大类子类定义典型项目
一致性成本预防成本防止缺陷发生培训、质量计划、供应商评价
鉴定成本检测评估是否合格检验、测试、校准
非一致性成本内部故障交付前发现缺陷返工、报废、停工
外部故障交付后发现缺陷退货、保修、索赔

1:10:100法则

预防阶段花1元解决的问题，鉴定阶段需10元，外部故障阶段需100元。预防成本是最有效的投资。

质量成本口诀预防鉴定是好事(一致性)，内外故障是坏事(非一致性)；预防最值钱，外部最赔钱。
分类陷阱
  • 返工→内部故障成本
  • 客户投诉处理→外部故障成本
  • 供应商评价→预防成本
  • 计量器具校准→鉴定成本
  • 质量培训→预防成本

2.1 SIPOC模型 工具

SIPOC是过程改进的高层次流程图，在Define阶段用于界定项目范围。

字母英文中文说明
SSupplier供方提供输入的组织或个人

PProcess过程将输入转化为输出的活动
OOutput输出过程的结果

实例：餐厅点餐SIPOCS:食材供应商、顾客 | I:食材、点餐信息 | P:接单->备料->烹饪->上菜 | O:菜品、服务 | C:用餐客人

2.2 过程绩效指标 公式

RTY（流通合格率）

RTY公式
RTY = Y1 x Y2 x Y3 x ... x Yn
实例3道工序合格率95%、90%、98%。RTY = 0.95 x 0.90 x 0.98 = 83.79%

DPMO（百万机会缺陷数）

DPMO公式
DPMO = (缺陷总数 / (单位产品数 x 缺陷机会数)) x 1,000,000
实例1000块电路板，每块50个焊点，共200个缺陷。DPMO = (200/(1000x50)) x 10^6 = 4,000

FPY vs RTY vs 最终合格率

指标定义大小关系
FPY(首次合格率)首次通过过程(无返修)的合格率最小
RTY(流通合格率)各工序合格率乘积中间
最终合格率含返修后的合格率最大
关键易错点FPY <= RTY <= 最终合格率。RTY考虑了过程中所有工序的累积合格率。
2.3 水平对比(Benchmarking) 工具
类型对比对象特点
内部水平对比组织内部同类最佳部门最容易实施
竞争性水平对比直接竞争对手数据难获取
功能性水平对比不同行业但功能相似可突破行业思维
一般性水平对比不限行业最佳实践范围最广

实施步骤

  1. 确定对比主题和关键过程
  2. 选择标杆对象
  3. 收集数据(自身和标杆)
  4. 分析差距和原因
  5. 制定改进计划并实施
2.4 价值链与价值流分析 工具

价值链（波特）

  • 基本活动(5个)：内部物流->生产运营->外部物流->市场营销->售后服务
  • 辅助活动(4个)：采购、技术开发、人力资源管理、企业基础设施

价值流图(VSM)中的三类活动

类型定义策略
增值活动(VA)客户愿意付费优化提升
必要非增值(NNVA)不增值但当前必须做减少/简化
非增值活动(NVA)纯浪费消除
考试要点七大浪费记忆：TIMWOOD(Transport运输、Inventory库存、Motion动作、Waiting等待、Over-processing过度加工、Overproduction过量生产、Defects缺陷)。

项目管理

3.1 项目选择原则与方法 方法

选择原则

  • 与战略对齐：支撑组织战略方向
  • 有意义：对客户和财务有显著影响
  • 可度量：有明确基线和目标
  • 可管理：范围适当，3-6个月可完成
  • 可支持：有高层支持和资源保障

项目选择方法

方法说明
帕累托分析识别造成80%问题的20%关键原因
平衡计分卡从四个维度评估项目价值
约束理论(TOC)围绕系统瓶颈做项目

收益-难度矩阵

  • 高收益、低难度：优先做(Quick Win)
  • 高收益、高难度：重点投入
  • 低收益、低难度：有空再做
  • 低收益、高难度：不做
3.2 SMART目标与立项表 工具
字母英文中文含义
SSpecific具体的目标清晰明确

AAchievable可实现的有挑战性但可达成

立项表(Project Charter)核心要素

  1. 项目名称
  2. 问题描述(含量化基线数据)
  3. 目标陈述(SMART目标)
  4. 项目范围(SIPOC辅助)
  5. 预期收益

3.3 团队建设五阶段(塔克曼模型) 概念
阶段特征领导者角色
1.形成期(Forming)互相认识，规则不明，依赖领导者明确目标、角色和规则
2.震荡期(Storming)冲突出现，质疑目标和分工处理冲突，重申目标
3.规范期(Norming)规则建立，开始合作鼓励参与，适当授权
4.执行期(Performing)高效运转，自主解决冲突授权、支持、移除障碍
5.解散期(Adjourning)项目结束，总结经验庆祝成果，知识沉淀
团队五阶段口诀形成震荡规范执行解散：认识->吵架->立规矩->干活->散伙
3.4 沟通管理与风险管理 方法

沟通渠道数

公式
渠道数 = n x (n-1) / 2
实例10人团队沟通渠道 = 10 x 9 / 2 = 45条

风险管理四步

  1. 风险识别：列出可能影响项目的风险
  2. 风险评估：评估概率(P)和影响(I)
  3. 风险优先级：风险值 = P x I
  4. 风险应对（四策略）
风险应对四策略
  • 规避：改变计划消除风险
  • 转移：转给第三方(如买保险)
  • 减轻：降低概率或影响
  • 接受：接受不采取行动
3.5 甘特图与关键路径法(CPM) 工具

甘特图

横轴为时间，纵轴为任务，条形显示各任务开始和结束时间。

关键路径法

  • ES：最早开始时间
  • EF：最早完成 = ES + 工期
  • LS：最晚开始 = LF - 工期
  • LF：最晚完成时间
  • 总浮动 = LS - ES = LF - EF

关键路径上的任务浮动时间为0，不能延误，否则影响整体进度。

界定(Define)

VOC客户声音、CTQ树、卡诺模型、QFD质量屋、问题陈述、Is-Is Not分析
4.1 VOC客户声音与CTQ 概念

VOC收集方法

方法类型优缺点
问卷调查被动覆盖面广但深度有限
深度访谈主动深入了解但样本小
焦点小组主动群体互动激发想法

现场观察主动发现隐性需求

CTQ树（需求分解）

将客户需求逐层分解为可度量的技术指标：

CTQ树实例客户需求：上菜快 -> 驱动因素：点餐响应、备菜、上菜时间 -> CTQ指标：点餐后30分钟内上菜(可度量)
考试要点VOC->CTQ转化路径：客户需求(模糊)->驱动因素->CTQ(可度量规格)。CTQ必须可度量。
4.2 卡诺模型(Kano Model) 工具
类型满足时不满足时策略
1.基本型需求(Must-be)不增加满意度极度不满必须满足
2.期望型需求(One-dimensional)满意不满持续提升
3.兴奋型需求(Attractive)惊喜不不满创新差异化
4.无差异型(Indifferent)无影响无影响减少投入
5.反向型(Reverse)不满满意避免
卡诺模型口诀基本型：有它不喜，没它生气；期望型：越多越好，线性关系；兴奋型：有它惊喜，没它不怪。
动态演变兴奋型->期望型->基本型。今天令客户惊喜的功能，明天可能成为基本期望。需求持续创新。
4.3 质量功能展开(QFD) 工具

将客户需求系统转化为产品设计要求的方法论，核心工具是质量屋(House of Quality)。

质量屋结构

部分内容
左墙(Whats)客户需求(VOC)及重要度
天花板(Hows)技术特性/工程措施
房间(关系矩阵)需求与技术特性的关系(强9/中3/弱1)
屋顶(相关矩阵)技术特性间的相关性(正/负)
右墙(竞争评估)自身与竞争对手对比
地下室(技术评估)技术重要度、目标值

QFD四阶段展开

  1. 产品规划：客户需求->产品技术特性
  2. 零件配置：产品技术特性->零件特性
  3. 工艺设计：零件特性->工艺要求
  4. 生产控制：工艺要求->生产要求
考试要点关系矩阵通常用9-3-1表示强-中-弱关系。技术特性重要度 = Sigma(需求重要度 x 关系权重)。
4.4 问题陈述与Is-Is Not分析 方法

问题陈述五要素

  • 什么：出了什么问题
  • 在哪里：问题发生地点
  • 何时：问题开始时间
  • 多大：量化程度(基线数据)
  • 影响：对客户和业务的影响
问题陈述实例2026年Q1，XX产品退货率从去年同期2.1%升至4.8%，主要集中在华东地区，主要原因是外观划伤，每月额外损失约15万元。

Is-Is Not分析

维度Is(是)Is Not(不是)可能差异原因
什么外观划伤不是尺寸超差包装/搬运工序
哪里华东地区不是华北/华南物流线路
何时Q1开始不是Q4以前Q1有什么变化

测量(Measure)

数据类型、描述性统计、正态分布、过程能力(Cp/Cpk/Pp/Ppk)、测量系统分析(MSA)
5.1 数据类型与测量尺度 概念

数据类型

类型定义举例统计方法
连续型(计量型)可取任意值长度、重量、时间均值、t检验、ANOVA
离散型(计数型)只能取特定值合格/不合格、缺陷数比例、卡方检验

四种测量尺度

尺度性质运算举例
定类(Nominal)分类无序=, !=性别、颜色
定序(Ordinal)有序间距不等=, !=, <, >评级(优良中差)
定距(Interval)等间距无绝对零+, -温度(C)
定比(Ratio)等间距有绝对零+, -, x, /重量、金额
易错点
  • 温度是定距不是定比(0度C不代表没有温度)
  • 定序数据不能计算均值
  • 连续型数据比离散型信息量更大
5.2 描述性统计 公式

集中趋势

指标公式特点
均值x-bar = Sigma(xi)/n受极端值影响大
中位数排序后中间值不受极端值影响
众数出现最多的值可能不唯一

离散程度

指标公式说明
极差(Range)R = max - min只用了两个数据
方差(样本)s^2 = Sigma(xi - x-bar)^2 / (n-1)样本方差分母用n-1
标准差s = sqrt(s^2)与数据同量纲
变异系数(CV)CV = (s / x-bar) x 100%可跨量纲比较
样本标准差(无偏估计)
s = sqrt[ Sigma(xi - x-bar)^2 / (n - 1) ]
关键易错点总体方差分母用N，样本方差分母用n-1(贝塞尔校正)。n很大时差别很小。

偏度与峰度

  • 偏度：=0对称，>0右偏(正偏)，<0左偏(负偏)
  • 峰度：=0正态，>0尖峰(厚尾)，<0平峰(薄尾)
5.3 正态分布与中心极限定理 概念

经验法则(68-95-99.7)

范围包含比例
mu +/- 1sigma~68.27%
mu +/- 2sigma~95.45%
mu +/- 3sigma~99.73%
mu +/- 6sigma~99.9999998%

Z变换

Z = (X - mu) / sigma
实例mu=100mm, sigma=2mm, LSL=94mm。Z=(94-100)/2=-3，P(Z<-3)=0.135%

中心极限定理(CLT)

无论总体什么分布，样本量足够大(n>=30)时，样本均值分布趋近正态，均值=mu，标准差=sigma/sqrt(n)。

标准误差
sigma(x-bar) = sigma / sqrt(n)
考试要点sigma未知时用s代替。样本量越大，样本均值变异越小。
5.4 过程能力分析(Cp/Cpk/Pp/Ppk) 公式
指标公式条件含义
Cp(USL-LSL)/(6 x sigma-hat)受控+正态，短期不考虑偏移的潜在能力
Cpkmin[(USL-mu)/(3sigma-hat), (mu-LSL)/(3sigma-hat)]受控+正态，短期考虑偏移的实际能力
Pp(USL-LSL)/(6s)不要求受控，长期长期潜在性能
Ppkmin[(USL-x-bar)/(3s), (x-bar-LSL)/(3s)]不要求受控，长期长期实际性能
Cp公式
Cp = (USL - LSL) / (6 x sigma-hat) = 公差带宽 / 过程波动范围
Cpk公式
Cpk = min[CPU, CPL]，其中 CPU=(USL-mu)/(3sigma-hat), CPL=(mu-LSL)/(3sigma-hat)
实例USL=10.5, LSL=9.5, mu=10.0, sigma-hat=0.1
Cp=(10.5-9.5)/(6x0.1)=1.67; CPU=(10.5-10.0)/(3x0.1)=1.67; CPL=(10.0-9.5)/(3x0.1)=1.67; Cpk=min(1.67,1.67)=1.67(均值居中，Cp=Cpk)

Cp与Cpk关系

Cpk = Cp x (1 - k)，其中 k = |mu - M| / ((USL-LSL)/2)

M=(USL+LSL)/2为规格中心。均值居中时k=0，Cpk=Cp。

Cpk评价标准

Cpk评价sigma水平
<1.0不足<3sigma
1.0~1.33勉强可接受3-4sigma
1.33~1.67良好4-5sigma
1.67~2.0优秀5-6sigma
>=2.0六西格玛6sigma
关键易错点
  • Cpk <= Cp 永远成立
  • Cp/Cpk用sigma-hat(来自控制图R-bar/d2)，短期受控
  • Pp/Ppk用样本标准差s，长期
  • 计算前必须先验证受控+正态
记忆技巧Cpk x 3 = 短期Z值；短期Z - 1.5 = 长期Z。如Cpk=1.33->短期Z=4->长期Z=2.5->DPMO~6,210
5.5 测量系统分析(MSA) 方法

测量系统五大特性

特性定义合格标准
1.偏倚(Bias)测量值与基准值的系统差异偏倚<=10%公差带宽
2.线性(Linearity)偏倚随量程变化的程度斜率接近0
3.稳定性(Stability)随时间的变化控制图无异常
4.重复性(Repeatability, EV)同一操作者多次测量同一零件的变异%EV<10%优秀
5.再现性(Reproducibility, AV)不同操作者测量同一零件的变异%AV<10%优秀

Gage R&R评价

%Gage R&R = (6 x sigma测量系统 / 公差带宽) x 100%
%Gage R&R评价建议
<10%优秀可接受
10%~30%有条件接受取决于应用重要性
>30%不可接受必须改进

NDC(分辨力)

NDC = 1.41 x (PV / Gage R&R)，NDC>=5为合格
MSA五性口诀偏线稳重再：偏倚、线性、稳定性、重复性、再现性。前三性看准确性，后两性看精密度。
关键易错点
  • 重复性=同一人多次测(设备变异)
  • 再现性=不同人测(操作者变异)
  • 计算过程能力前必须先做MSA
  • MSA在Measure阶段进行

6.1 假设检验基础 概念
原假设H0备择假设H1
含义无差异、无效果有差异、有效果
目的想要推翻的想要证明的
符号=, <=, >=!=, >, <

两类错误

H0为真H0为假
接受H0正确(1-alpha)第二类错误(beta) 存伪
拒绝H0第一类错误(alpha) 弃真正确(1-beta) 检验功效
alpha = P(拒绝H0 | H0为真) — 显著性水平，通常0.05
beta = P(接受H0 | H0为假) — 存伪概率
Power = 1 - beta — 检验功效，越大越好(通常>=0.80)
高频考点
  • 第一类错误(alpha)：弃真=生产者风险=没病说有病
  • 第二类错误(beta)：存伪=消费者风险=有病说没病
  • alpha和beta不能同时减小(样本量固定时)
  • 增大样本量可同时减小两者
  • p值 < alpha -> 拒绝H0；p值 >= alpha -> 不拒绝H0
两类错误口诀alpha弃真(生产者风险)：没问题但判不合格，生产者吃亏；beta存伪(消费者风险)：有问题没查出，消费者吃亏。

6.3 方差分析(ANOVA) 公式
ANOVA分解
SS总 = SS组间 + SS组内
F = MS组间 / MS组内 = (SS组间/(k-1)) / (SS组内/(N-k))
来源SSdfMS
组间SSB=Sigma ni(xi-xbar)^2k-1MSB=SSB/(k-1)
组内SSW=Sigma Sigma(xij-xi)^2N-kMSW=SSW/(N-k)
总计SST=Sigma Sigma(xij-xbar)^2N-1-

k=组数，N=总样本数

三前提假设

  • 正态性：各组近似正态
  • 方差齐性：各组方差相等(Bartlett/Levene检验)
  • 独立性：观测值相互独立

事后多重比较(Post-hoc)

方法特点
Tukey法最常用，控制整体误差率
Bonferroni法最保守，alpha/比较次数
Dunnett法各组与对照组比较
Fisher LSD最不保守，可能假阳性
易错点ANOVA的H0是"所有组均值相等"，H1是"至少一组不同"(不是"所有组都不同")。拒绝H0后需事后比较确定具体差异。
6.4 回归分析 公式

简单线性回归

y-hat = b0 + b1*x
最小二乘法
b1 = Sxy / Sxx = Sigma(xi-xbar)(yi-ybar) / Sigma(xi-xbar)^2
b0 = ybar - b1*xbar

判定系数R^2与相关系数r

R^2 = SSR/SST = 1 - (SSE/SST)，0<=R^2<=1
r = Sxy / sqrt(Sxx * Syy)，-1<=r<=1，r^2=R^2
|r|范围强度
0.0~0.2极弱/无
0.2~0.4弱
0.4~0.6中等
0.6~0.8强
0.8~1.0极强

残差分析四假设

  • 独立性：残差相互独立
  • 正态性：残差服从正态分布
  • 等方差性：残差方差恒定(无喇叭形)
  • 线性：Y与X是线性关系(无弯曲趋势)
关键易错点
  • 相关不等于因果
  • 回归中自变量X的误差忽略不计
  • 多元回归必须用调整R^2(Adjusted R^2)
  • R^2总随自变量增加而增大，调整R^2可能减小
6.5 卡方检验 公式

拟合优度检验

chi^2 = Sigma[(Oi - Ei)^2 / Ei]

独立性检验(列联表)

RPN(风险优先数)

RPN公式
RPN = S x O x D
参数含义范围高分含义
S(Severity)严重度：失效对客户的影响1-1010=极严重(安全/法规)
O(Occurrence)发生度：失效发生的概率1-1010=几乎必然发生
D(Detection)检测度：到达客户前被发现概率1-1010=几乎不可能检测
RPN实例S=8, O=4, D=3。RPN=8x4x3=96。RPN越高优先处理，通常RPN>100需优先行动。
易错点
  • D(检测度)评分方向与S和O相反：D分越高表示越难检测(风险越大)
  • S(严重度)由客户决定
  • FMEA是预防性工具，在问题发生前使用
RPN口诀SOD三数相乘得RPN：S严重(客户感受)、O发生(出现频率)、D检测(发现能力，越难越高分)

术语含义举例

7.3 DOE效应分析 公式

主效应

效应A = y-bar(A+) - y-bar(A-)

因子A在高水平(+)的平均响应减去低水平(-)的平均响应

交互效应

效应AB = 1/2[(y-bar(A+B+)-y-bar(A-B+)) - (y-bar(A+B-)-y-bar(A-B-))]
考试要点如果存在显著交互效应，不能单独解释主效应。AB交互显著时，A的效应取决于B的水平。

分析方法

  • 帕累托图：按效应绝对值排序，超参考线的显著
  • 正态概率图：偏离直线的点为显著效应
  • 主效应图：展示各因子对响应的影响方向和大小
  • 交互效应图：两条线不平行表示有交互效应

7.6 SMED(快速换模) 工具

由丰田工程师新乡重夫提出，意为10分钟以内换模。

核心思想：区分内部与外部换模

类型定义策略
内部换模必须停机才能进行转为外部换模
外部换模不停机也能进行提前做好

SMED四步

  1. 区分：区分内部和外部操作
  2. 转化：将内部转为外部
  3. 并行：内部操作并行化
  4. 简化：消除调整，使用快速紧固

7.7 约束理论(TOC) 方法

由高德拉特提出，核心：任何系统至少存在一个约束(瓶颈)，系统产出受瓶颈限制。

TOC五步聚焦法

  1. 识别：找到系统瓶颈
  2. 利用：充分利用瓶颈产能
  3. 服从：非瓶颈服从瓶颈节奏
  4. 突破：增加瓶颈产能
  5. 回到：解决后找新瓶颈
TOC五步口诀找到瓶颈->利用瓶颈->服从瓶颈->突破瓶颈->找新瓶颈

三大指标

指标含义目标
产出率(Throughput, T)销售额-原材料成本最大化
库存(Inventory, I)投资在可销售物上的资金最小化
运营费用(OE)将库存转为产出的花费最小化

类型功能举例

9.1 DFSS概述与DMADOV流程 概念

DFSS定义

DFSS(Design for Six Sigma) = 六西格玛设计，用于新产品/新过程的设计阶段。DMAIC用于改进已有过程，DFSS用于从零开始设计。

9.2 田口稳健设计(Taguchi Robust Design) 方法

田口方法核心思想

由田口玄一提出，核心：质量是设计出来的，不是检验出来的。通过参数设计使产品对噪声因素（不可控因素）不敏感。

三类因素

类型定义控制策略
信号因子希望影响的输入(设定值)调整到目标值
控制因子设计者可控的参数优化选择最佳水平
噪声因子不可控的环境/制造变异通过控制因子使其影响最小化

三阶段设计

  1. 系统设计：确定基本技术方案和架构
  2. 参数设计（核心）：选择控制因子最佳水平组合，使产品对噪声不敏感——这是田口方法的精髓
  3. 容差设计：在参数设计基础上，确定各参数的公差范围，平衡质量和成本

信噪比(S/N Ratio)

望目特性(目标值最优)
S/N = -10 x log10[ (1/n) x Sigma(yi - y0)^2 ]
望大特性(越大越好)
S/N = -10 x log10[ (1/n) x Sigma(1/yi^2) ]
望小特性(越小越好)
S/N = -10 x log10[ (1/n) x Sigma(yi^2) ]
实例某设计有望目特性，目标值y0=10。3次试验结果：9.8, 10.1, 9.9。S/N = -10 x log10[(1/3)x((9.8-10)^2+(10.1-10)^2+(9.9-10)^2)] = -10 x log10[(1/3)x(0.04+0.01+0.01)] = -10 x log10[0.02] = 16.99 dB
关键概念
  • 望目：有特定目标值(如尺寸=10mm)
  • 望大：越大越好(如强度、寿命)
  • 望小：越小越好(如噪声、磨损)
  • S/N越大越好，表示产品越稳健
  • 参数设计优先于容差设计(先优化参数再定公差)
田口方法口诀质量是设计出来的；参数设计是精髓；先参数后容差；信噪比越大越稳健。
9.3 容差设计(Tolerance Design) 公式

容差设计原则

  • 在参数设计完成后进行
  • 权衡质量损失与制造成本
  • 对关键参数收紧公差，非关键参数放宽公差
  • 使用容差分析确定各参数对输出的贡献

容差叠加（极值法与统计法）

极值法(最坏情况)
T总 = T1 + T2 + ... + Tn
统计法(概率法，假设正态分布)
T总 = sqrt(T1^2 + T2^2 + ... + Tn^2)
实例3个零件叠加，各公差±0.1mm。
极值法：T总 = 0.1+0.1+0.1 = ±0.3mm
统计法：T总 = sqrt(0.01+0.01+0.01) = sqrt(0.03) = ±0.173mm
统计法更经济(公差更紧)，但要求各零件独立正态分布。

田口质量损失函数

L(y) = k x (y - m)^2

L=损失，y=实际值，m=目标值，k=损失系数。偏离目标值就有损失，越偏离损失越大(二次函数)。

易错点传统观点认为"在规格限内就没有损失"。田口认为偏离目标值就有损失，即使合格也有损失。这是田口质量观与传统质量观的根本区别。
9.4 TRIZ(发明问题解决理论) 方法

定义

TRIZ = "发明问题解决理论"(俄语缩写)，由阿奇舒勒提出，基于对数百万专利的分析，总结出发明的规律。

核心概念

  • 矛盾(Contradiction)：改善某参数会导致另一参数恶化
  • 40个发明原理：解决技术矛盾的通用方法
  • 39个工程参数：描述矛盾的两端
  • 矛盾矩阵：根据矛盾查找推荐的发明原理
  • 理想最终结果(IFR)：系统自己解决问题，不增加复杂度

四种矛盾类型

矛盾类型定义解决工具
技术矛盾改善参数A导致参数B恶化矛盾矩阵+40原理
物理矛盾同一参数需要两个相反值分离原理(空间/时间/条件)
实例雨伞：需要大(遮雨)又需要小(便携)——物理矛盾。解决：折叠伞(条件分离——展开时大，收起时小)。
考试要点TRIZ常考概念：40个发明原理、39个工程参数、矛盾矩阵、IFR(理想最终结果)。核心是消除矛盾而非折中。
9.5 Pugh矩阵(概念选择) 工具

定义

Pugh矩阵是多方案概念选择工具，在DFSS的Analyze阶段用于选择最优设计方案。

使用方法

  1. 确定评估准则(来自客户需求和CTQ)
  2. 选择一个方案作为基准(Datum)
  3. 其他方案与基准比较：优(+)、相同(S)、差(-)
  4. 统计各方案+和-的数量
  5. 总分 = +(数) - -(数)，分最高者为优
Pugh矩阵实例
准则方案A(基准)方案B方案C
成本D+-
性能DS+
可靠性D-+
制造性D+S
总分(+-)0+1+1

方案B和C并列，需进一步细化准则或加权后决策。

9.6 可靠性与维修性基础 公式

可靠性定义

产品在规定条件和时间内完成规定功能的概率。

核心指标

指标公式含义
可靠度R(t)R(t) = P(T > t)t时刻仍在正常工作的概率
失效率lambdalambda = 失效数 / (总数 x 时间)单位时间失效概率
MTBFMTBF = 1 / lambda平均故障间隔时间
MTTFMTTF = 1 / lambda平均失效前时间(不可修)

浴盆曲线(Bathtub Curve)

阶段特征失效率变化
1.早期失效期制造缺陷、运输损伤高但快速下降
2.偶然失效期正常使用中的随机失效低且恒定
3.耗损失效期材料老化、磨损快速上升
指数分布可靠度(恒定失效率)
R(t) = e^(-lambda x t) = e^(-t/MTBF)
实例某设备MTBF=1000小时，求运行500小时的可靠度：R(500) = e^(-500/1000) = e^(-0.5) = 0.6065 = 60.65%
考试要点MTBF用于可修复产品，MTTF用于不可修复产品。指数分布假设失效率恒定(浴盆曲线中间段)。串联系统可靠度 = R1 x R2 x ... x Rn，并联系统可靠度 = 1-(1-R1)(1-R2)...(1-Rn)。
